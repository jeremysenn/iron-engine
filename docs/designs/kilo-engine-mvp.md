## Vision

### 10x Check
The 10x version isn't more features — it's the KILO rule engine becoming the CANONICAL
implementation of the methodology. If KILO Strength Society themselves would point coaches
at this software and say "this is how you apply what you learned," that's 10x. That requires
100% rule accuracy validated by KILO instructors and a rule engine so good that KILO
incorporates it into their curriculum.

## Architecture Note
The full product follows Approach B from DESIGN.md (KILO Rule Engine + LLM Coaching Layer).
The MVP ships the rule engine only (Layer 1 + Layer 2). The LLM layer (Layer 3) is the
first post-MVP release. "Show your work" annotations use structured rule references, not LLM.

MVP is a Rails 8 full-stack app using ERB + Hotwire (Turbo Frames/Streams). No separate
frontend, no API-only mode. For MVP, only coaches have accounts (User model with auth).
Clients are data-only records (Client model, no login).

## Success Criteria (MVP is done when)
- A KILO-trained coach can enter a client's PrimeEight E1RMs, training age, goal, and
  frequency, and receive a complete 12-week KILO-compliant program in under 5 minutes.
- Every generated program passes the seed data verification test suite (576 periodization
  rows match the source PDFs exactly).
- At least one program generated by the engine matches a manually-built program from the PDFs
  (same exercises, rep schemes, phase assignment, and training method timing for identical inputs).
- A coach can record MAP scores for all 14 movement patterns and the engine filters exercise
  selection accordingly (e.g., failed level 2 squat -> goblet squat instead of barbell back squat).

## Accepted Scope (added to this plan)

MVP ships the deterministic rule engine (Layers 1-2 from DESIGN.md). LLM layer ships post-MVP.

**Core MVP scope:**
- User + auth (Rails 8 built-in)
- Client model
- PrimeEight Assessment + Strength Ratio Calculator
- All KILO reference data models + seed data
- All 8 service objects as defined in DESIGN.md: map_assessment_engine, strength_ratio_calculator,
  periodization_engine, macrocycle_builder, training_split_selector, training_method_timer,
  session_generator, program_generator
- Program + Macrocycle + Mesocycle + Microcycle + TrainingSession + SessionExercise + ExerciseSet
- Program display view

**Cherry-picked additions:**
- MAP Assessment models, UI, and engine integration. Seed data scope: 14 movement patterns,
  3-4 levels each, ~50 assessment rows + ~200 progression rows (see DESIGN.md seed data strategy).
- Strength ratio radar chart: Rendered as SVG on the assessment results page. Plots all 8
  PrimeEight ratios vs optimal. Upper body chart: Bench Press (100%), OH Press (72%),
  Incline (91%), Dip (117%), Chin-Up (87%). Lower body chart: Squat (100%), Front Squat (85%),
  Deadlift (125%). Limiting lift highlighted in red. Updates per assessment record.
  No interactivity for MVP. Effort cap: if SVG chart takes more than 2 days, ship a plain
  HTML table of ratios instead and defer the chart.
- "Show your work" annotations: Displayed as collapsed annotation panels on the program show
  page, one per decision point (model selection, limiting lift ID, phase assignment, training
  method timing). Triggered by a "Show reasoning" toggle. Rule name + logic string only.
- Macrocycle timeline view: After generating a program, a coach can see a 12-week phase
  calendar to verify the periodization at a glance before sharing with a client. Rendered as
  a 12-column HTML table (one per week), rows for phase, training method, and session count.
  Color-coded by phase (accumulation/intensification). Static display only.
- Exhaustive seed data verification test suite (576 periodization rows + strength ratio examples).

**Seed data strategy:** Transcribe from KILO PDFs into Google Sheets/CSV first. Import via
a Rake task (`rake seed:from_csv`) that reads CSVs from `db/seed_data/*.csv` and upserts
reference records. This allows non-developer verification (a KILO coach can review the
spreadsheet) and makes corrections trivial. Estimated effort: 1-2 weeks of manual
transcription + verification. This is the highest-risk task.

**Seed data mitigation:** To unblock engineering, transcribe a representative subset first
(e.g., Model 2.2 Intermediate Medium Volume, one goal, one frequency = ~48 rows). The
engine can be built and tested against the subset while full transcription continues in
parallel. This prevents the seed data effort from blocking all other work.

**E1RM formula:** KILO uses a rep intensity percentage lookup table (not the Lander equation).
The DESIGN.md glossary example (weight / rep_intensity_percentage) reflects KILO's approach.
The Lander equation mentioned as "ported from existing app" is a fallback for when the rep
count falls outside the KILO lookup table. When rep count falls outside the table, the engine
falls back to Lander and adds a warning annotation to the output. The engine must use KILO's
table as primary. Verify against the Program Design Resource PDF.

## Pre-Implementation Blockers
- **Ruby/Rails version**: Currently Ruby 2.0.0 / Rails 4.1.8. Need Ruby 3.3+ and Rails 8.
  Install via rbenv or asdf, then `gem install rails` and regenerate Kapp scaffold with
  `rails new Kapp --database=postgresql`. This must happen before any implementation.
- **KILO IP licensing**: Not yet formalized. Building for personal coaching practice first.
  Initiate the conversation with KILO Strength Society before starting Phase 2 (assessments).
  Does not need to be resolved, but must be started and response received or two weeks
  elapsed before investing in the methodology-specific seed data transcription.
  If licensing is denied, the methodology-specific seed data must be removed. Personal
  coaching use is likely permissible; get legal opinion before any commercial launch.
