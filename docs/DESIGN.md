## Problem Statement

ShapeTrak is a fitness coaching SaaS with users but no paying customers. The founder (a KILO-trained coach) wants to differentiate by encoding the complete Kilo Strength Society methodology as an opinionated AI engine. Competing platforms (TrueCoach, TrainHeroic, CoachRx) offer generic AI workout generation. None encode KILO's specific system of MAP assessments, PrimeEight exercises, optimal strength ratios, limiting lift identification, undulating periodization, and training method timing. The goal is to be the only software that thinks like a KILO coach.

## Demand Evidence

- The founder is a KILO-trained coach who uses ShapeTrak for their own coaching practice and currently references 12+ PDF documents to manually look up progressions, rep schemes, macrocycle structures, and training method timing
- KILO Strength Society has an active community of trained coaches who all use the same methodology but have no software purpose-built for it
- Coaches currently spend hours per client manually cross-referencing PDFs to build periodized programs... the same deterministic lookup that software can do in milliseconds

## Status Quo

Coaches trained in the KILO methodology currently:
1. Run MAP assessments on paper/spreadsheets and manually determine movement levels
2. Test PrimeEight E1RMs and hand-calculate strength ratios to identify limiting lifts
3. Look up the correct macrocycle structure from the Optimizing Strength Ratios PDF based on limiting lift combinations
4. Cross-reference the Periodization Resource for rep schemes, intensity percentages, and volume ranges based on training age and goal
5. Select training splits from the Training Split Database PDF
6. Time training methods (IMCA, eccentrics, isometrics, speed-strength) to the correct accumulation/intensification phases
7. Manually build out 12-week programs with all the above applied correctly

This process takes 2-4 hours per client program and is error-prone because the decision tree has dozens of branches. One wrong lookup (e.g., using a novice intensity fluctuation for an intermediate trainee) compounds through the entire macrocycle.

## Target User & Narrowest Wedge

**Target user:** A strength coach who has completed KILO's Foundation curriculum (Program Design, Periodization, Optimizing Strength Ratios courses) and coaches 5-20 clients. They understand the methodology deeply but lose hours to manual program construction.

**Narrowest wedge:** Enter a client's PrimeEight E1RMs and training age. ShapeTrak instantly calculates strength ratios, identifies the limiting lift, and generates the correct 12-week macrocycle with all sessions, exercises, rep schemes, tempos, rest periods, and training methods pre-filled according to KILO specifications. One click replaces 2-4 hours of PDF cross-referencing.

## Constraints

- KILO materials are copyrighted. The software encodes the logic (which the coach has licensed through their coursework) but does not reproduce the educational content verbatim
- The rule engine must be 100% accurate to KILO specifications. A wrong rep scheme or incorrect strength ratio erodes all trust
- LLM explanations must be clearly labeled as AI-generated and must never contradict the deterministic rule engine output

## Key Decision: Fresh Rails 8 Application

The existing ShapeTrak app runs Rails 6.1 (EOL October 2024) with legacy patterns (CoffeeScript, jQuery, Turbolinks 5, denormalized column-per-set schema). Since there are no paying customers and the KILO engine is the entire value proposition, the decision is to build a fresh Rails 8 application purpose-built around the KILO methodology.

**What ports over from the existing app (logic, not files):**
- Body composition calculations (Jackson-Pollock 3/7-site, BMI, waist-hip ratio, priority fat loss sites)
- E1RM calculation (Lander equation)
- Stripe subscription/billing integration logic
- Client management and invitation flows
- Email notification patterns

**What gets redesigned from scratch:**
- Data model designed around KILO's PrimeEight, MAP assessments, and periodization structures
- Normalized exercise session schema (not column-per-set)
- Modern Rails 8 stack: Hotwire (Turbo Frames/Streams), built-in authentication, Solid Queue
- All views and controllers purpose-built for the KILO coaching workflow

## Premises

1. This is NOT generic AI workout generation. ShapeTrak will encode the complete KILO methodology as a rule-based expert system. Given MAP assessment results, PrimeEight E1RMs, training age, goal, and frequency, there is ONE correct program according to KILO. The AI's job is to apply the rules perfectly every time.
2. The data pipeline is: MAP Assessment (movement quality) -> Strength Ratio Analysis (limiting lifts) -> Periodization Model Selection -> Training Split -> Rep Schemes -> Training Methods -> Complete Program. Each step is deterministic per KILO's specifications.
3. The existing ShapeTrak codebase provides proven business logic (body comp, E1RM, billing) that ports to the new app, but the workout programming layer is replaced entirely by the KILO engine.
4. The target customer is a KILO-trained coach who understands the methodology but spends hours manually looking up the right progressions, rep schemes, and macrocycle structures from these PDFs. ShapeTrak automates the lookup and generation.
5. No other platform encodes KILO's methodology. This is not competing on "AI" ... it's competing on being the only software that thinks like a KILO coach.

## Approaches Considered

### Approach A: KILO Rule Engine Only
Pure algorithmic KILO logic encoded in Rails services. No LLM. Every decision is deterministic and testable. Strengths: 100% accurate, no API costs, millisecond generation. Weaknesses: no natural language explanations, doesn't help coaches communicate "why" to clients.

### Approach B: KILO Rule Engine + LLM Coaching Layer (CHOSEN)
Same deterministic rule engine as A for all program generation. Adds an LLM layer that: (1) explains decisions in natural language ("Your front squat ratio is 80%, optimal is 85%. This is your lower body limiting lift. Here's why we're prioritizing front squat in Intensification phases..."), (2) generates client-facing program summaries, (3) answers methodology questions. The rule engine is the source of truth; the LLM is the translator.

### Approach C: LLM-First with KILO Knowledge Base
Feed all KILO PDFs to an LLM and let it generate programs through prompting. Rejected because KILO's specifications are too precise for an LLM to consistently reproduce. A hallucinated rep scheme (e.g., 5x8 at 83% when KILO specifies 5x7 at 80%) would destroy coach trust.

## Recommended Approach

**Approach B: KILO Rule Engine + LLM Coaching Layer**

The rule engine handles the math and the rules. The LLM handles the words. Neither does the other's job.

### KILO Domain Glossary (for engineers)

- **PrimeEight**: The 8 primary barbell exercises: Squat, Front Squat, Deadlift, Overhead Press, Incline Press, Bench Press, Dip, Chin-Up
- **E1RM**: Estimated 1-Rep Max. Calculated as: weight / rep_intensity_percentage (e.g., 100lbs for 5 reps: 100 / 0.85 = 118 E1RM)
- **Optimal Strength Ratios**: Target percentages between PrimeEight lifts. Lower body: Squat 100%, Front Squat 85%, Deadlift 125%. Upper body: Bench Press 100%, OH Press 72%, Incline 91%, Dip 117%, Chin-Up 87%
- **Limiting Lift**: The exercise with the largest negative discrepancy from its optimal ratio. One is identified per body region (upper + lower)
- **90-Degree Principle**: Exercise pairing rule. Within a microcycle, pressing exercises rotate by 90 degrees (OH Press -> Bench Press, Incline -> Dip). Within a session, the primary press is paired with its 90-degree assistance (OH Press -> Flat DB Press, Bench -> Seated DB Press)
- **Volume Tolerance**: Low, Medium, or High. Determines which periodization sub-model to use (e.g., Model 2.1 vs 2.2 vs 2.3 for Intermediate). Based on coach's assessment of client recovery capacity
- **Periodization Model**: Numbered system. First digit = training level (1=Novice, 2=Intermediate, 3=Advanced). Second digit = volume (1=Low, 2=Medium, 3=High). Each model specifies exact rep schemes for 16 macrocycles across 4 years
- **Macrocycle**: 12 weeks. Contains 4 mesocycles (Accumulation 1, Intensification 1, Accumulation 2, Intensification 2)
- **Mesocycle**: 3 weeks. Each contains 3 microcycles
- **Microcycle**: 1 week. Contains 2-4 training sessions depending on frequency
- **Accumulation phase**: Higher volume, lower intensity. Reps >= previous accumulation
- **Intensification phase**: Lower volume, higher intensity. Reps <= previous intensification
- **Training Methods**: Concentric (IMCA, Speed-Strength), Eccentric (Sub-Maximal, Supra-Maximal), Isometric (Yielding, Overcoming). Timed to specific phases based on training level and goal
- **MAP Assessment**: Movement Analysis & Progressions. Multi-level movement quality tests (squat, deadlift, split squat, etc.) with pass/fail at each level and specific exercise progressions per outcome

### Data Pipeline Flow

```
1. Coach enters client MAP assessment results
   -> map_assessment_engine determines movement levels per pattern
   -> Suggests exercise progressions for failed movements
   -> Validation: all 6 lower body + 8 upper body patterns assessed

2. Coach enters PrimeEight E1RMs (or they're calculated from existing sessions)
   -> strength_ratio_calculator computes ratios vs optimal
   -> Identifies upper body + lower body limiting lifts
   -> Calculates lift goals
   -> Validation: minimum 4 of 8 PrimeEight lifts required (Squat + Bench mandatory,
      these are the 100% reference lifts). Missing lifts flagged with "assess before 
      generating" warning. Partial generation not allowed.

3. Coach selects: goal (Hypertrophy/Abs Str/Rel Str/Power), 
                  frequency (2x/3x/4x/week), 
                  volume tolerance (Low/Medium/High)
   -> periodization_engine selects model (e.g., Model 2.2 = Intermediate Medium Volume)
   -> macrocycle_builder creates 12-week structure with correct phases
   -> training_split_selector picks the right split per phase
   -> training_method_timer assigns methods to each phase

4. program_generator orchestrates all above
   -> Creates Program with 4 Mesocycles
   -> Creates TrainingSessions for each day in each Microcycle
   -> Creates SessionExercises with position (A1/A2/B1/B2/C1/C2)
   -> Creates ExerciseSets with target reps and weights
   -> Applies 90-degree principle for exercise pairing
   -> Selects grip types/widths per the chin-up/rowing selection rules

5. LLM coaching_explainer generates (async, non-blocking):
   -> Program overview ("Here's why this macrocycle targets your front squat...")
   -> Per-phase rationale ("Accumulation 1 uses IMCA at 70% because...")
   -> Client-ready summary for the coach to share
```

### Seed Data Strategy

The KILO knowledge base needs to be seeded into the database. This is a significant manual effort:

- **9 periodization models** (1.1 through 3.3) with 16 macrocycles x 4 phases each = **576 rows** of rep scheme data. Each row: model_id, macrocycle_number (1-16), phase (ACC/INT), rep_scheme string, calculated intensity_pct
- **8 optimal strength ratios** (one per PrimeEight exercise)
- **~40 macrocycle structure templates** for limiting lift combinations (from Optimizing Strength Ratios PDF, pages 5-13)
- **Training method taxonomy**: ~20 methods with category, tempo, and description
- **MAP assessment criteria**: 14 movement patterns x 3-4 levels = ~50 assessment rows + ~200 progression rows
- **Exercise database**: scope TBD pending full review of KILO Exercise Database PDF
- **Training split templates**: ~30 templates across goals, phases, and training levels

**Estimated seeding effort:** 1-2 weeks of manual transcription + verification. Each row must be verified against the source PDF. Verification approach: seed the data, then write a test suite that generates known programs (e.g., "Intermediate, Hypertrophy goal, Front Squat limiting lift") and compares output against manually-built programs from the PDFs.

### Testing Strategy

The rule engine's "100% accuracy" requirement demands thorough testing:

1. **Seed data verification tests**: For each periodization model, assert that rep scheme at macrocycle N, phase P matches the PDF. All 576 rows verified.
2. **Strength ratio calculation tests**: Known inputs (e.g., Squat E1RM 450, Front Squat E1RM 360) produce exact expected ratios and correct limiting lift identification. Test cases from the Optimizing Strength Ratios PDF examples.
3. **Program generation integration tests**: For each goal x training level combination, generate a full program and verify: correct model selected, correct macrocycle structure, correct training methods timed to correct phases, 90-degree principle applied, grip selections correct.
4. **Edge case tests**: All 8 E1RMs identical (no limiting lift), extreme ratios (200% deadlift), novice with only bodyweight assessments.
5. **LLM explanation regression tests**: Cache known-good explanations. Flag when regenerated explanations contradict the rule engine output (e.g., explanation says "Accumulation" but rule engine assigned "Intensification").
